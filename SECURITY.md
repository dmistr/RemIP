# Security Policy

## Supported Versions

Only the latest stable version of Remip receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.98.x  | :white_check_mark: |
| < 0.98  | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Remip, please report it privately.

### How to report

1. **Do not** create a public GitHub issue
2. Send an email to: **dmistr9999@gmail.com**
3. Include:
   - Version of Remip you're using
   - Windows version
   - Steps to reproduce the issue
   - Any relevant logs or screenshots

### What happens next

- Your report will be acknowledged
- The issue will be investigated
- You will be notified when a fix is ready
- You will be credited for the discovery (unless you prefer to remain anonymous)

### What is considered a security vulnerability

- Remote code execution
- Privilege escalation
- Authentication bypass
- Unauthorized access to saved credentials
- Information disclosure of sensitive data

### What is NOT considered a security vulnerability

- Bugs that do not have security implications
- UI/UX issues
- Feature requests
- Issues requiring physical access to the machine

Thank you for helping keep Remip secure!
